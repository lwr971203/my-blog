    列表初始化

    # 列表初始化

    # 传统的术语

    一般的：

    • 对于使用()或者是{}直接放在变量后面的方式，称为直接初始化；
    • 对于将字面量通过配合=号或者其他方式进行传递给变量的方式，称为复制初始化
    • 复制初始化不一定是调用的复制构造函数，只是给这个初始化的形式起了一个特定的名字而已，很多时候还是调用的直接初始化所调用的构造函数
    struct Demo {
    Demo(int a) {}
};

int main() {
    int i1(1); // 直接初始化
    int i2 = 2; // 复制初始化

    Demo d1(3); // 直接初始化
    Demo d2 = 4; // 复制初始化，但实际上调用的还是直接初始化所调用的构造函数
    Demo d3 = Demo(5); // 复制初始化，但实际上调用的还是直接初始化所调用的构造函数
}
    • 使用new操作符，以及类的构造函数初始化列表进行初始化的方式，仍然被归类为直接初始化；函数传参，return返回值则是复制初始化
    struct Demo {
    Demo(int a) : b(10.1) { // 直接初始化

    }

    double b;
};

void foo(Demo) {}

Demo get() {
    return 2;
}

int main() {
    Demo* d1 = new Demo(1); // 直接初始化
    foo(5); // 复制初始化
    Demo d = get(); // 复制初始化
}

    # 使用列表初始化

    • 调用构造函数
    #include <string>

struct Demo {
    Demo(std::string a, int b) {}
    Demo(int a) {}
};

void foo(Demo) {}

Demo get() {
    return {"world", 5};
}

int main() {
    int x = {5}; // 复制初始化
    int x1{8}; // 直接初始化

    Demo x2 = {4}; // 复制初始化
    Demo x3{2}; // 直接初始化

    foo({8}); // 复制初始化
    foo({"hello", 8}); // 复制初始化

    Demo x4 = get(); // 复制初始化
    Demo* x5 = new Demo{"hi", 42}; // 直接初始化
}
    • 对容器进行初始化
    #include <vector>
#include <list>
#include <set>
#include <map>
#include <string>

int main() {
    int x[] = {1, 2, 3, 4, 5};
    int x1[]{1, 2, 3, 4, 5};

    std::vector<int> x2{1, 2, 3, 4, 5};
    std::vector<int> x3 = {1, 2, 3, 4, 5};

    std::list<int> x4{1, 2, 3, 4, 5};
    std::list<int> x5 = {1, 2, 3, 4, 5};

    std::set<int> x6{1, 2, 3, 4, 5};
    std::set<int> x7 = {1, 2, 3, 4, 5};

    // 解释：内层调用的是std::pair(const T1& x, const T2& y)；外层调用的是std::map(std::initializer_list<value_ type> list, const Allocator&)
    std::map<std::string, int> x8{
        {"bear", 4}, {"cassowary",2}, {"tiger",7}
    };
    std::map<std::string, int> x9 = {
        {"bear", 4}, {"cassowary",2}, {"tiger",7}
    };
}

    对容器进行初始化本质上是对{}中的元素产生一个initializer_list对象，initializer_list对象就是一个拥有begin、end以及size成员函数的类模板

    # 指定初始化

    • 在c++20中，在使用列表初始化的时候可以指定数据成员
    struct Point {
    int x;
    int y;
    int z;
};

int main() {
    Point p1{.x = 4, .y = 2}; // 对于没有指定的数据成员，例如z，则自动使用默认初始化为0
}
    • 指定初始化的条件：
      • 必须是一个聚合体：①数组；②class、struct、union且没有自定义构造函数，没有基类，没有虚函数，数据成员全部public
      • 指定的数据成员必须是非静态的
      • 每个非静态数据成员最多只能初始化一次
      // Point p{ .x = 1, .x = 2}; // error
      • 指定初始化时，不能乱序，只能按照声明的顺序，注：可以跳过
      • 不能指定数据成员的数据成员进行初始化，也就是不能嵌套指定，但是可以通过一定的手法进行指定：例如
      Line l1{ .start{ .x = 10} }
      • 联合体的数据成员总共只能初始化一次，并且指定初始化不需要按照顺序
      union u {
  int a;
  const char* b;
};

int main() {
  u u1{ .a = 1 };
  u u2{ .b = "abcd" };
  // u u3{ .a = 1, .b = "abcd" }; // error
}

    # 细节

    • 列表初始化既可以调用匹配的构造函数，也可以调用以std::initializer_list为参数的构造函数
    • 列表初始化的多种形式的优先级为：聚合初始化、具有std::initializer_list参数的构造函数 > 匹配的构造函数 > 默认初始化
    • 对于std::initializer_list这个类模板，可以直接使用列表初始化！，所以具有std::initializer_list参数的函数，都可以直接用({})调用
    • 列表初始化在很多时候对于收窄转换是禁止的，在编译期直接报错。当然具体还是看使用的什么编译器

    学前必读 c++11
